컬렉션인터페이스


컬렉션 프레임워크: 데이터 집합을 효율적으로 관리하기 위해 표준화 시킨
-종류는 콜렉션(list,set),map이 있다.
-콜렉션은 하나의 값을 이용
-맵은 두개의 값을 이용
-list : 순서 ,중복허용
-set: 순서 보장하지않음 ,중복불가
-map : key,value로 이루어짐 , key는 중복불가 ,value는 중복가능 , id와 pw를 생각하면됨
-ArrayList : list 인터페이스를 구현한 클래스
-ArrayList의 객체를 생성할 때 어떤 데이터(자료형 또는 클래스) 를 이용할 건지 결정해 주어야한다.


list,set,map
list : 순서 O 중복 O
set : 중복 X 순서 X
map: <key,value> :  key:중복X     value: 중복O

중복체크는 어떻게?
1.해당 객체의 equals를 확인
2.없으면 부모 클래스의 equals를 확인

==>> 클래스 선언 시 equals를 오버라이딩 하지 않으면 객체의 중복 확인을 주소로 확인한다.
==>> 필요하면 클래스 선언 시 equals를 오버라이딩 해야한다.

스택: 역순        		메소드호출할때         //스택은 클래스가있다.
큐: 순서대로		메시지 큐 , 프린터 출력관리 //큐는 인터페이스

일반적으로 인터페이스는 객체를 생성할 수 없지만
해당 인터페이스를 구현한 클래스를 이용하여 객체를 생성할 수 있다.
